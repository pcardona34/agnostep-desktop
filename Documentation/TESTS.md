# Tests protocol

    To report a successfull test on a new hardware, please report to us the following steps:

Date: Passed: (sum of the successful steps) / 19 (percent)

- First Installation on a fresh SD-card (Pi Systems) or bootkey, step by step: 1 prep | 2 wmaker | 3 GNUstep | 4 Frameworks | 5 Core apps/extra/devel/games/utilities | 6 user settings | 7 Display manager
- Window manager / Theming
- Workspace / Dock / Wallpaper
- Home customized: Folders names and Icons
- Applications Folder with a shortcut Icon on the Shelf / TabbedShelf
- Autostart: namely conky panel and notifications (Birthday or feast): modify the GlinGlin date in the conf file.
- Autostart: namely Classic flavour with Clip and Dock
- Sysinfo Panel (if a laptop, battery monitoring is showing)
- Dockapps (if a laptop, batmon)
- Notify: Updater with action, BirthNotify and associated tools
- Colors changing within wmtext: orange for birthday, crimson for upgradable packages.
- Run successfully Upgrade wrapper to update the Debian system
- Open/save Panel with the home shortcut available
- Web Browser: openURL service: text link selected, link clicked from a message within GNUMail
- Printing / set default printer using Cups from 'Printer.app' wrapper
- Scanning (with ScanImage)
- Screensaver (InnerSpace)
- Display Manager (LightDM): login, shutdown.
- Mount a removable device: with wmudmount in the classic flavour.
- Unmount with DND on the Recycler desktop icon (classic) or on the docked icon (conky)
- Listening to music (with Cynthiune) and setting the level (with VolumeControl)
- Watching a video (with Player)
- Theming of FocusWriter, Firefox...
